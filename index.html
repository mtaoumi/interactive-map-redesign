Case study

Before / After

A walk through the reasoning behind every redesign decision.

Before
  • Default red pins create visual noise; nothing is hierarchically distinct.
  • No filtering — users must scan every marker to find a category.
  • Search hidden in a hamburger; controls scattered across the screen.
  • Popups are unstyled and crop content awkwardly on mobile.
After
  • Category-coded markers act as a visual legend embedded in the map itself.
  • Persistent sidebar pairs map and list — two views, one mental model.
  • Search and filter chips live above the fold, always reachable.
  • Popups become editorial cards with title, category, and a photo slot.

Visual hierarchy

Every element earns its weight on screen. Display type carries the title, body type carries the substance, and only one accent green is allowed to compete for attention at a time.

Information architecture

Eight places, four categories. Filtering reduces working memory load — users don't need to remember which pins are cafés; the system carries that for them.

Spatial design

The layout reads like a plan: a defined edge (sidebar), a public realm (map), and a transition zone (header). Spatial logic from architecture, applied to a screen.

Accessibility

Skip links, ARIA roles on tabs and lists, keyboard-reachable filters, AA-contrast type, and a focus-visible ring on every interactive element.
